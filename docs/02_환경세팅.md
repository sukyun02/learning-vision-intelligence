# 환경세팅 계획서 (Linux 메인 · Windows 부록)

**프로젝트**: PyramidNet-272 α200 + ShakeDrop · CIFAR-100
**대상 환경**:
- 🖥️ **메인**: Linux 서버 (학습용) + Linux 노트북 (외장 GPU, 코드 개발 + 학습)
- 💻 **보조**: Windows 노트북 (코드 수정만, 학습 안 함)

> 원본 계획서가 Ubuntu/RTX 5070 Ti/ROS2 Docker 단일 시나리오를 가정했던 것과 달리, 실제로는 다중 머신 환경. 이 문서는 두 시나리오를 모두 다룬다.

---

## A. Linux (메인 학습 환경)

### A-0. 사전 점검

#### A-0-1. NVIDIA 드라이버 확인

```bash
nvidia-smi
```

확인 항목:
- **Driver Version**: PyTorch cu121 기준 → **525 이상** 권장
- **CUDA Version**: 12.1 이상
- **GPU 모델명** 표시 확인

> 이번 프로젝트는 [requirements.txt](../requirements.txt)에 명시된 **`torch==2.5.1+cu121`** 기준이다. cu128로 바꿀 이유는 없다 (Blackwell이 아닌 한).

#### A-0-2. (선택) ROS2 컨테이너와 GPU 공유 시

학습 중 GPU VRAM 충돌을 방지하려면:

```bash
docker stop adas_container   # ROS2 컨테이너 종료
docker ps                    # 실행 목록에서 사라졌는지 확인
```

학습이 끝나면 `docker start adas_container`로 복구. **이 단계는 GPU를 ROS2와 공유하는 머신에만 적용된다.**

#### A-0-3. Python 버전

```bash
python3 --version    # 3.10 이상 권장
sudo apt install python3-venv python3-pip git -y
```

### A-1. 프로젝트 클론 & venv

```bash
git clone <repo-url> cifar100-pyramidnet
cd cifar100-pyramidnet

python3 -m venv venv
source venv/bin/activate
which python   # 출력: <project>/venv/bin/python
```

### A-2. 의존성 설치

```bash
pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

[requirements.txt](../requirements.txt) 핵심 패키지:
- `torch==2.5.1+cu121`, `torchvision==0.20.1+cu121`
- `wandb==0.25.1`, `python-dotenv==1.2.2`
- `tqdm`, `matplotlib`, `numpy`, `pillow`

> 학교 서버 등 cu121이 호환 안 되는 환경이면 PyTorch만 다른 cu 버전으로 별도 설치 후 나머지를 `pip install -r requirements.txt --no-deps`로 처리.

### A-3. 환경 변수 설정 — `.env`

```bash
cp .env.example .env
nano .env
```

[.env](../.env) 권장 값 (VRAM 절약):

```ini
WANDB_API_KEY=<본인 키 — 절대 커밋 금지>
WANDB_PROJECT=pyramidnet-cifar100
WANDB_ENTITY=<본인 엔티티>

SEED=42
EPOCHS=1000
BATCH_SIZE=64       # VRAM 절약 (256→64)
LR=0.05             # PyramidNet 원논문 reference (bs=128, lr=0.1)에서 linear scaling
WEIGHT_DECAY=0.0001 # 5e-4→1e-4 (ShakeDrop 원논문 값)
SWA_EPOCHS=250
NUM_WORKERS=6

DATA_ROOT=./data
CKPT_DIR=./checkpoints
```

> ⚠️ **WANDB API key 회전 권고**: 현재 [.env](../.env)에 평문으로 저장되어 있고 사용자가 외부 도구와 공유한 이력이 있다. https://wandb.ai/authorize 에서 키를 재발급하고 .env를 갱신하라.

### A-4. PyTorch GPU 확인

```bash
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

기대 출력:
```
2.5.1+cu121
True
NVIDIA GeForce RTX <모델>
```

### A-5. 검증 스모크 테스트

#### A-5-1. 1-epoch 스모크 (VRAM·파이프라인 확인)

```bash
python train.py --epochs 1
```

확인:
- VRAM 점유 (`nvidia-smi -l 1` 별도 터미널)가 ~8-12GB 이내인지
- WandB run이 생성되는지
- `checkpoints/log_seed42.csv` 생성

#### A-5-2. (선택) 단일 배치 overfit 디버그

> 향후 [train.py](../train.py)에 `--mode overfit_batch` 플래그 추가 예정 (코드개발계획 Phase E).

### A-6. WandB 로그인

```bash
wandb login   # API key 붙여넣기
```

또는 `.env`의 `WANDB_API_KEY`를 [train.py](../train.py:33)의 `load_dotenv()`가 자동 인식하므로 건너뛰어도 됨.

## B. Windows 노트북 (보조 — 코드 수정만)

> 학습은 안 한다. 문법 검사, 단위 테스트, 커밋만 수행.

### B-1. Python & Git Bash

- Python 3.10+ 설치 (https://www.python.org/downloads/windows/)
- Git Bash 또는 PowerShell

### B-2. venv

```bash
# Git Bash
python -m venv venv
source venv/Scripts/activate

# 또는 cmd
venv\Scripts\activate.bat
```

### B-3. CPU-only PyTorch 설치 (학습 안 함)

```bash
pip install --upgrade pip
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt --no-deps
pip install wandb python-dotenv tqdm matplotlib numpy pillow
```

> CUDA 설치 불필요. import 검증과 1-epoch 작은 모델 dry-run 정도만 가능.

### B-4. 동작 확인

```bash
python -c "import torch; from models.pyramidnet import pyramidnet272; m = pyramidnet272(num_classes=100); print(sum(p.numel() for p in m.parameters())/1e6, 'M')"
```

기대 출력: `28.2 M` 정도.
